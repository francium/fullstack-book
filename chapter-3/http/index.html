Chapter 3 HTTP

Hypertext Transfer Protocol (HTTP) is the primary protocol web browsers and web servers use to communicate with each other. If you're writing web applications HTTP will be the center piece of everything. HTTP is an application layer protocol and generally abstracts away all details from the lower layers of the OSI network stack.

When you open a web browser and try to go to a website, for example, http://example.com, the web browser will connect to the web server that's responsible for serving the webpages and other resource for the website. This all happens using the HTTP protocol. The web browser will issue an initial request encoded according to the HTTP specification and the web browser will know how to decode and understand the request. Similarly, when the server responds to the request, it will encode a response according to the HTTP specification which the web browser will know how to decode and understand.

HTTP is a request-response protocol and stateless: a request is made and responses are expected, and there is no protocol level association between different request-response pairs[1]. HTTP requests and responses are primarily encoded in plaintext, but can include binary data[2].
